闪购新店初始分拆分

关联：闪购综合分拆分（范围收窄）
业务反馈：新店加权后排名断崖式下跌
改动范围：管理后台（新增Tab）+ 数据组（ES脚本）

📋 需求背景

闪购新店目前使用与外卖同一套「外賣新店初始分數配置」。新店移除加权后排名断崖式下跌，业务反馈「新店加权开咗都排到5/6页后」。

范围收窄：原计划全量拆分闪购综合分（含独立排序配置、5个混排场景），开发评估改动量大，本次只拆分「闪购新店初始分」。

核心目标：建立闪购独立的新店初始分体系。加权期内获合理曝光，结束后平稳过渡。
🏗️ 整体方案

参考外卖初始分机制，初始分作为额外加分，与自然综合分独立计算。

闪购新店复用外卖初始分机制，在此基础上增加按分类的额外加分，即：

初始分总分 = 百分比得分（由「取二级品类近7日分数百分比」决定） + 分类加分（由「主營加分 / 輔營加分」决定）

新店总分 = 自然综合分（含「新门店」维度，可在门店综合排序配置中调整） + 初始分总分

全部参数均为可配置项，默认值与外卖一致：

配置项（管理后台字段名）默认值说明
取二级品类近7日分数百分比75%决定百分比得分的基准
二级品类门店数小于10百分比15%品类门店数<10时的兜底
持续天数24天初始分最多持续多久
按分数衰退百分比30%衰退触发的品类排名阈值
衰退天数18天每日衰退 = 初始分总分 ÷ 衰退天数
每日衰退分值 = 初始分总分（百分比得分 + 分类加分） ÷ 衰退天数，两者同步线性衰减，每日减固定分值，直至归零或到期。

衰退触发逻辑：

  • 若「開啟新店/老店加權時」选择「開始衰退」：加权即刻开始线性衰退，不等待分数条件
  • 若选择「不處理」：需等自然综合分+初始分总分 ≥ 品类第「按分数衰退百分比」位的门店分时，才开始衰退
  • 无论哪种模式，初始分最多持续「持续天数」天，到期归零

生效条件：门店需完成至少1单。

加分优先级（品类加分）：优先取门店所属二级分类的加分；若二级未设置（0或未填），向上取一级分类；均未设置则为0。

主营/辅营取值规则：主营加分与辅营加分累加计算，两者同时生效。

🎨 交互原型

管理后台：業務運營 → 搜索管理 → 綜合排序配置。新增頂層Tab「閃購新店初始分數配置」，平行於「外賣新店初始分數配置」。内容：新店初始分字段 + 品類額外加分表格。

📋 字段规格

新店初始分（7个字段，与外卖一致）

字段默认值校验
*初始化分数→取二级品类近7日分数百分比75%1-100
*二级品类门店数小于10百分比15%1-100
*持续天数24天小于50正整数
*按分数衰退→品类百分比30%1-100
*衰退天数18天小于50正整数
*开启新店/老店加权时开始衰退开始衰退/不处理
*歇业保护是否保留天数是是/否
